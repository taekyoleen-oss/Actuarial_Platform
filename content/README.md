# 콘텐츠 스테이징 폴더

앱이 라이브(.env·DB 적용·관리자 계정)되기 전, 게시할 자료(PDF 등)를 여기에 두면
스크립트가 **Supabase Storage 업로드 + `ib_posts` 게시물 생성**까지 처리합니다.

## 어디에 두나
카테고리별 하위 폴더에 둡니다.

| 폴더 | 카테고리 | slug |
|------|----------|------|
| `content/exclusive-rights/` | 보험 배타적 사용권 분석 | `exclusive-rights` |
| `content/global/` | 해외 주요 보험 정보·자료 | `global` |
| `content/domestic/` | 국내 보험 정보·분석 | `domestic` |

→ **보험 배타적 사용권 분석 자료는 `content/exclusive-rights/` 에 올려주세요.**

## 어떻게 두나 (택1)
1. **PDF만**: 그냥 PDF 파일을 넣습니다. 제목 = 파일명(확장자 제외), 본문은 비움(나중에 관리자 화면에서 보완 가능).
2. **PDF + 사이드카(권장)**: PDF와 같은 이름의 `.md` 파일을 함께 넣어 제목·본문을 지정합니다.

   예) `2024-배타적사용권-분석.pdf` + `2024-배타적사용권-분석.md`

   사이드카 `.md` 형식:
   ```markdown
   ---
   title: 2024년 보험 배타적 사용권 분석
   author: 홍길동
   published: true
   ---

   여기에 게시물 본문(요약·해설)을 작성합니다. AI 요약과 별개로 표시됩니다.
   ```

## 그 다음
파일을 두신 뒤 알려주시면:
- Supabase 자격증명(anon/service_role 키)이 준비된 경우 → 제가 import 스크립트로 Storage 업로드 + 게시물 생성을 실행합니다.
- 아직 키가 없으면 → 스크립트만 준비해두고, 키 확보 후 1회 실행하면 일괄 등록됩니다.

> 운영 단계에서는 이 폴더 대신 **관리자 UI(`/admin/posts/new` → 수정 화면에서 PDF 첨부)**로 올리면 됩니다. 이 폴더는 초기 일괄 등록·백업용입니다.

> 이 폴더의 실제 파일은 git에 커밋하지 않습니다(용량·민감도). `.gitignore` 참조.
