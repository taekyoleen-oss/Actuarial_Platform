# Exclusive Rights Summary — 배타권 보고서 4단 요약

보험 신상품 **배타적사용권** 분석 보고서의 AI 요약을, 아래 고정 4단 포맷으로 **에이전트가 직접 작성**한다.
LLM API를 호출하지 않는다(웹 관리자 "AI 생성" 경로만 API 사용 — 동일 양식).

## 양식 단일 출처

요약 양식의 정본은 **`lib/summary-format.ts`의 `EXCLUSIVE_RIGHTS_SUMMARY_PROMPT`**.
웹 API 경로(`lib/summarize.ts`)와 이 스킬(수동 작성)이 같은 양식을 공유한다.
**이 문서의 4단 포맷을 바꾸면 `lib/summary-format.ts`도 반드시 함께 수정한다(드리프트 방지).**

## 4단 포맷 (고정 순서·제목)

```markdown
## 총평
- 보고서 핵심 결론 2~4불릿 (개조식·체언 종결, 핵심어 **굵게**)
- 배타적사용권 기간·종료일 명시:
  '**배타적사용권 N개월** · YYYY.MM.DD 부여 → **YYYY.MM.DD 만료**'
  (일자 불명 → 'YYYY.MM 부여 → YYYY.MM 만료', 부여정보 없음 → '부여(기간·종료일 미상)')
- 판매량·점유율·순위 등 변동성 수치 제외

## 급부구조
- 본문 급부/지급구조 '표'의 각 행을 불릿으로 나열
- 지급사유 · 지급금액 · 대상/코드 · 한도 · 보장기간 등 항목별

## 의의
- 기존 보장·서비스와의 차별점·장점 위주 2~4불릿
- 신규성의 의의('업계 최초', 보장 공백 점유 등)

## 제안
- 원문 제안(확장·집중·전환·재구성 등) 프레임별 1불릿, 이해하기 쉽게 풀어서
```

공통: `-`불릿 + 체언 종결 + `**굵게**` + `##`섹션. **추정 점수·평점·등급 수치 절대 금지**(예: "약 78/100", "독창성 20·진보성 35"). 원문에 없는 내용 추측 금지.

## 워크플로우 (보고서 1건당)

1. **원천 덤프** — `node scripts/dump-source.mjs "<제목 일부>"`
   → 제목·본문(`content`)·현재 첨부 PDF 추출 텍스트 출력. 이 텍스트만 근거로 작성.
2. **배타 일자 확인** — `output/exclusive_dates.json`에서 해당 보고서의 `granted_on`·`duration_months`·`expires_on` 확인.
   - JSON에 기간이 `null`이면 PDF 본문의 '배타권 현황/N개월 부여' 문구에서 보강하고, 부여일(심의일)+기간으로 종료일을 계산해 JSON을 갱신.
   - 종료일 = 부여일 + 기간. 일자 미상이면 월 단위. 둘 다 없으면 '미상'.
   - '심의 중·부여 미정'(예: 하나손보 변호사선임)은 총평에 그대로 '배타적사용권 심의 중(부여 미정)'으로 정직하게 표기.
3. **요약 작성** — 4단 포맷으로 직접 작성해 `_workspace/summary_<키>.md`에 저장.
   급부구조는 반드시 본문 '표'를 행 단위로 옮긴다.
4. **DB 기록** — `node scripts/set-summary.mjs "<제목 일부>" _workspace/summary_<키>.md`
   (4섹션 존재·점수문구 미포함을 스크립트가 가드. 통과 시 `ib_posts.summary` 갱신, 무API.)
5. **검증** — `node scripts/set-summary.mjs "<제목 일부>" --check`로 저장본 확인.
   QA: ①4섹션 존재 ②점수문구 0 ③배타 종료일(또는 미상) 표기 ④급부구조가 표 기반.

## 헬퍼

| 스크립트 | 역할 |
|---|---|
| `scripts/dump-source.mjs` | 제목·본문·현재 PDF 텍스트 출력(작성 입력) |
| `scripts/set-summary.mjs` | 작성한 마크다운을 `ib_posts.summary`에 기록(양식 가드, 무API) |
| `output/exclusive_dates.json` | 배타 부여(심의)일·기간·종료일 캐시 |
| `lib/summary-format.ts` | 양식 정본(API·스킬 공유) |

## 원칙

- **무(無)API**: 요약 생성에 Anthropic API를 쓰지 않는다. 에이전트가 원천을 읽고 작성한다.
- **정직성**: 부여일/종료일/기간이 확인 안 되면 추정하지 말고 '미상'으로 표기.
- **표 기반 급부구조**: 급부구조는 창작이 아니라 본문 표의 전사(轉寫).
- **확장**: 향후 '해외자료 요약'·'국내보험 정보 요약'은 별도 스킬로 추가하되, 본 4단 포맷과 `lib/summary-format.ts` 구조를 기준점으로 재사용한다.
